1.0 Wprowadzenie

Microsoft® SQL Server™ Compact 4.0 to zajmujący niewiele miejsca na dysku wewnątrzprocesowy aparat bazy danych, który umożliwia deweloperom tworzenie niezawodnych aplikacji dla komputerów stacjonarnych z systemem Windows oraz witryn zaprojektowanych za pomocą pakietu startowego ASP.NET.

3.0 Znane problemy

3.1

Macierzyste biblioteki DLL programu SQL Server Compact 4.0 wymagają obecności bibliotek wykonawczych programu Microsoft Visual C++ 2008 (x86 i x64) z dodatkiem Service Pack 1. W przypadku instalacji programu SQL Server Compact 4.0 za pomocą pliku instalatora instalowane są również biblioteki wykonawcze programu Visual C++ 2008 z dodatkiem SP1. Jeśli program SQL Server Compact 4.0 jest wdrażany prywatnie w folderze aplikacji, w celu zapewnienia prawidłowego działania tego programu na komputerze muszą być zainstalowane następujące aplikacje:

  1. Instalacja programu .NET Framework 3.5 z dodatkiem SP1 powoduje zainstalowanie bibliotek wykonawczych programu Visual C++ 2008 z dodatkiem SP1.

  2. Biblioteki wykonawcze programu Visual C++ 2008 z dodatkiem SP1 można pobrać i zainstalować z poniższej lokalizacji:
    Visual C++ 2008

Należy zauważyć, że instalacja programu .NET Framework 2.0, .NET Framework 3.0 lub .NET Framework 4 nie powoduje zainstalowania bibliotek wykonawczych programu Visual C++ 2008 (x86 i x64) z dodatkiem Service Pack 1.

3.2

W przypadku uruchomienia plików wykonywalnych instalatora programu SQL Server Compact na komputerze, na którym nie zainstalowano programu .Net Framework, instalacja odbywa się bez błędów, jednak niektóre kroki są pomijane. Może to spowodować wystąpienie błędów na takim komputerze. Pomijane są następujące kroki:

  • Wpis w pliku machine.config programu .NET nie będzie uwzględniał programu SQL Server Compact jako dostawcy danych.

  • Odpowiednie zestawy plików wykonywalnych programu Compact nie są umieszczane w pamięci Global Assembly Cache.

Może się tak również zdarzyć, gdy użytkownik wykona następujące czynności:

  • Zainstalowanie pliku wykonywalnego programu SQL Server Compact 4.0

  • Zainstalowanie programu .NET Framework 3.5 z dodatkiem SP1 lub .NET Framework 4.0

W celu rozwiązania tego problemu należy ponownie zainstalować/naprawić pliki wykonywalne programu SQL Server Compact po zainstalowaniu programu .NET Framework.

3.3

Program SQL Server Compact 4.0 nie rozpoznaje właściwości Encrypt klasy SqlCeConnection, dlatego nie należy używać tej właściwości do szyfrowania plików bazy danych.

Uwaga:

Właściwość Encrypt została zastąpiona w programie SQL Server Compact w wersji 3.5. Właściwość ta została zachowana w programie SQL Server Compact 4.0 tylko w celu zapewnienia zgodności z poprzednimi wersjami.

W celu szyfrowania plików bazy danych programu SQL Server Compact 4.0 należy używać właściwości Encryption Mode klasy SqlCeConnection. Następujące przykłady ilustrują sposób korzystania z właściwości Encryption Mode.

  • Aby utworzyć nową zaszyfrowaną bazę danych programu SQL Server Compact 4.0:

    C#:

     
    SqlCeEngine engine = new SqlCeEngine("Data Source=Northwind.sdf;encryption mode=platform default;Password=<enterStrongPasswordHere>;");
engine.CreateDatabase();

    Visual Basic:

     
    Dim engine As SqlCeEngine = New SqlCeEngine("Data Source=Northwind.sdf;encryption mode=platform default;Password=<enterStrongPasswordHere>;")
engine.CreateDatabase()

  • Aby zmienić tryb szyfrowania istniejącej bazy danych programu SQL Server Compact 4.0:

    C#:

     
    SqlCeEngine engine = new SqlCeEngine("Data Source=Northwind.sdf;Password=<enterStrongPasswordHere>;");
engine.Compact("Data Source=Northwind.sdf;encryption mode= enginedefault;Password=<enterStrongPasswordHere>;");

    Visual Basic:

     
    Dim engine As SqlCeEngine = New SqlCeEngine("Data Source=Northwind.sdf;Password=<enterStrongPasswordHere>;") 
engine.Compact("Data Source=Northwind.sdf;encryption mode= engine default;Password=<enterStrongPasswordHere>;")

  • Aby zaszyfrować niezaszyfrowaną bazę danych programu SQL Server Compact 4.0:

    C#:

     
    SqlCeEngine engine = new SqlCeEngine("Data Source=Northwind.sdf");
engine.Compact("Data Source=Northwind.sdf;encryption mode=platform default;Password=<enterStrongPasswordHere>;");

    Visual Basic:

     
    Dim engine As SqlCeEngine = New SqlCeEngine("Data Source=Northwind.sdf;") 
engine.Compact("Data Source=Northwind.sdf;encryption mode=platform default;Password=<enterStrongPasswordHere>;"

3.4

Należy jawnie określić typy danych programu SQL Server Compact 4.0 dla parametrów takich jak SqlDbType i DbType.

Jeśli typy danych dla parametrów takich jak SqlDbType i DbType nie zostaną jawnie określone, zwracany jest wyjątek. W celu rozwiązania tego problemu należy jawnie określić typ danych dla parametrów takich jak SqlDbType i DbType. Ma to decydujące znaczenie w przypadku typów danych BLOB (obraz i ntext). Przykład kodu znajduje się poniżej:

C#:

 
SqlCeEngine engine = new SqlCeEngine(connString);
engine.CreateDatabase();
engine.Dispose();

SqlCeConnection conn = new SqlCeConnection(connString);
conn.Open();

SqlCeCommand cmd = conn.CreateCommand();
cmd.CommandText = "CREATE TABLE BlobTable(name nvarchar(128), blob ntext);";
cmd.ExecuteNonQuery();

cmd.CommandText = "INSERT INTO BlobTable(name, blob) VALUES (@name, @blob);";
SqlCeParameter paramName = cmd.Parameters.Add("name", SqlDbType.NVarChar, 128);
SqlCeParameter paramBlob = cmd.Parameters.Add("blob", SqlDbType.NText);
paramName.Value = "Name1";
paramBlob.Value = "Name1".PadLeft(4001);

cmd.ExecuteNonQuery();

Visual Basic:

 
Dim engine As SqlCeEngine = New SqlCeEngine(connString)
engine.CreateDatabase()
engine.Dispose()

Dim conn As SqlCeConnection = New SqlCeConnection(connString)
conn.Open()

Dim cmd As SqlCeCommand = conn.CreateCommand()
cmd.CommandText = "CREATE TABLE BlobTable(name nvarchar(128), blob ntext);"
cmd.ExecuteNonQuery() 

cmd.CommandText = "INSERT INTO BlobTable(name, blob) VALUES (@name, @blob);"
Dim paramName As SqlCeParameter
Dim paramBlob As SqlCeParameter
paramName = cmd.Parameters.Add("name", SqlDbType.NVarChar, 128)
paramBlob = cmd.Parameters.Add("blob", SqlDbType.NText)
paramName.Value = "Name1"
paramBlob.Value = "Name1".PadLeft(4001)

cmd.ExecuteNonQuery()

3.5

To wydanie programu SQL Server Compact 4.0 nie obsługuje synchronizacji danych z programem SQL Server za pomocą technologii takich jak Microsoft Sync Framework, replikacja scalająca czy zdalny dostęp do danych (RDA).

3.6

Generowanie modelu Entity Data Model dla programu SQL Server Compact nie powiedzie się, jeśli w schemacie programu wystąpią zduplikowane nazwy ograniczeń.

W programie SQL Server Compact nazwy ograniczeń są unikatowe w ramach tabeli, dzięki czemu mogą one być zduplikowane w bazie danych. Inaczej jest w przypadku programu SQL Server, gdzie nazwy ograniczeń są unikatowe w ramach bazy danych. Jeśli w schemacie programu SQL Server Compact występują zduplikowane nazwy ograniczeń integralności referencyjnej (relacja klucz podstawowy — klucz obcy), generowanie modelu Entity Data Model za pomocą kreatora tego modelu w programie ADO.NET Entity Framework nie powiedzie się. Problem można obejść, zmieniając nazwę zduplikowanej nazwy ograniczenia na nazwę unikatową w całej bazie danych, na przykład przez dodanie nazwy tabeli do nazwy ograniczenia.
